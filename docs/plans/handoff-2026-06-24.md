## Conventions / process (IMPORTANT — keep doing these)
- **CLAUDE.md rule #1:** get approval before user-affecting changes (API/schema/
  extension behavior/auth/pricing/rate). Don't push without an explicit OK + the
  deploy sanity-check. Live users — minimize impact, keep backward-compat, no DB
  migration unless reversible.
- **Branch workflow:** for substantive fixes, branch → fix → test → commit →
  `merge --ff-only` → push. Ali says "merge push" / "deploy et" to authorize.
- **Commit trailer (every commit):** `Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>`
- **Backend tests:** `cd backend && export JWT_SECRET=$(python -c "import secrets; print(secrets.token_hex(32))") && python -m pytest tests/<file> -q --no-header -p no:cacheprovider`. Run **specific files** — the FULL `pytest tests/` has a PRE-EXISTING JWT_SECRET collection quirk (NOT our bug). FakeSupabase (`tests/conftest.py`) ignores `.eq` filters, so model logic enforces visibility in Python; FastAPI TestClient DOES run BackgroundTasks (so async-send effects are testable).
- **Extension JS:** verify with `node --check extension/<file>.js`; validate i18n JSON with `node -e "JSON.parse(...)"`. i18n: `$NAME$` placeholder + a `placeholders: {name: {content: "$1"}}` map; the extension `t()` does positional substitution; ALWAYS keep a JS fallback string (i18n risk). - **Zip build (extension):** use the `.NET ZipArchive` PowerShell with forward-slash entry names (Windows `Compress-Archive` = BROKEN backslash zips that Chrome rejects). Events: `oauth_started/failed/completed`, `user_logged_in`, `oauth_retry` (new, only live once 0.1.17+ ships), `send_clicked/completed/failed`, `ext_installed/updated`, `$exception` (distinct_id `backend-server` / `extension-client`). `oauth_failed.reason` = `chrome_error` with `.message` ("The user did not approve access" = consent block; "Authorization page could not be loaded" = backend availability). ## Architecture quick-ref
- Extension MV3: `background.js` (service worker — OAuth via backend `/auth/login`, message handlers, token mgmt server-side), `sidebar.js` (~120KB — compose/send/CSV/quota/reports/feedback/settings), `popup.js`, `content_script.js` (FAB launcher injects sidebar), `config.js` (backend URL + PostHog EU key), `i18n.js`, `graph_api.js` (mostly VESTIGIAL — client-side OAuth/refresh replaced by backend).
- Backend FastAPI on Railway (web + worker + beat, shared code → a push redeploys all 3; web has healthcheck = zero-downtime, worker/beat brief restart, Celery `acks_late=True` so interrupted tasks recover; scheduled-send recovery via hourly `reset_stuck_sending_campaigns` sweep). `routers/{auth,campaigns,tracking,billing,settings,...}.py`, `workers/{scheduled_worker,followup_worker,celery_app}.py`, `models/`, `config.py`. Client ID/secret are **backend-only** (`AZURE_CLIENT_ID`/`SECRET` env). Send: `/send` validates synchronously then hands the recipient loop to a FastAPI BackgroundTask (`_run_campaign_send`); scheduled/followup sends run in Celery. `SEND_DELAY_SECONDS=2`.
